🛠️ Guide — Modifier mon site moi-même

Les Domaines de Tri Hent — mémo pratique pour Anne-Sophie

Le principe : votre site est fait de 4 fichiers texte. On modifie le texte à l'intérieur, on enregistre, et c'est en ligne. Le plus simple : éditer directement dans le Gestionnaire de fichiers de Hostinger (changement immédiat, sans rien re-téléverser).

1. Comment éditer un fichier (la méthode)

  1. Connectez-vous à hpanel.hostinger.com → votre site → Gestionnaire de fichiers
  2. Entrez dans le dossier public_html
  3. Clic droit sur le fichier voulu → « Modifier » (ou l'icône <>)
  4. Appuyez sur Ctrl + F pour chercher le texte à changer
  5. Modifiez, puis cliquez sur Enregistrer (icône disquette 💾 en haut à gauche)
  6. Rafraîchissez votre site (F5) → le changement est en ligne ⚡

Quel fichier pour quoi ?

FichierContenu
index.htmlPage d'accueil (accroche, services, bannière chiffres, témoignages, formules, FAQ, pied de page)
nos-biens.htmlLes biens de votre conciergerie
contact.htmlFormulaire + coordonnées
mentions-legales.htmlMentions légales

2. La règle d'or ✋

Ne jamais écrire le mot « gestion » (ni « gérer », « gère », « gérez »).
Ce terme est réservé aux détenteurs de la carte G. Utilisez plutôt : conciergerie, accompagnement, prise en charge, suivi, organisation.
Ne touchez qu'au texte, pas aux balises. Modifiez ce qui est entre les chevrons, jamais les <...> ni les class="...".
Exemple — dans <h3>Manoir de Kervelen</h3>, changez seulement Manoir de Kervelen.

3. Vos tâches courantes (exemples concrets)

➜ Corriger une phrase (ex. « à l'agence »)

Vous n'avez pas d'agence ? Dans contact.html, cherchez (Ctrl+F) : à l'agence

Rencontrons-nous autour d'un café, à l'agence ou chez vous.

Remplacez par :

Rencontrons-nous autour d'un café ou chez vous.

➜ Enlever la bannière des chiffres (« Biens accompagnés, taux d'occupation… »)

Dans index.html, cherchez class="chiffres". Sélectionnez et supprimez tout le bloc, depuis la ligne :

<section class="chiffres">

… jusqu'à la balise de fermeture correspondante :

</section>

(juste avant la section des témoignages). Enregistrez : la bannière disparaît. 💡 Astuce : faites d'abord une copie du fichier (voir §4) au cas où.

➜ Modifier / ajouter / retirer un bien

Tout se passe dans nos-biens.html. Chaque bien est un bloc qui commence par :

<div class="bien reveal">

… et se termine par le </div> qui lui correspond (juste avant le bien suivant).

Dans un bloc, ce que vous pouvez changer :

ÉlémentÀ chercher
Nom du bien<h3>...</h3>
Lieuclass="lieu"
Voyageurs / chambres / m²class="caracteristiques" (les <span>)
Descriptionclass="description"
Lien « Réserver sur Airbnb »href="https://www.airbnb.fr/" → remplacez par le lien réel de votre annonce Airbnb
Étiquette (Coup de cœur / Vue mer)class="badge" (supprimable)
Photossrc="https://images.unsplash.com/..." (voir §5)

Retirer un bien : supprimez tout son bloc, de <div class="bien reveal"> jusqu'à son </div>.

Ajouter un bien : copiez un bloc existant entier, collez-le juste après, puis modifiez le nom, le lieu, la description, les photos et le lien Airbnb.

4. Sécurité : ne rien casser 🛟

  • Avant une grosse modif, faites une copie du fichier : dans le Gestionnaire de fichiers, clic droit sur le fichier → Copier (vous aurez un index (copy).html de secours).
  • En cas d'erreur, une corbeille (« Trash bin ») existe dans le Gestionnaire de fichiers.
  • Filet de sécurité ultime : une sauvegarde complète du site existe sur GitHub (github.com/Aelwenor/domaines-trihent) et le fichier site-tri-hent.zip d'origine est sur votre ordinateur. En cas de pépin, on peut toujours tout restaurer.

5. Les photos (un peu plus technique)

Les photos actuelles viennent d'Internet (Unsplash) — ce sont des exemples. Pour mettre vos vraies photos plus tard, le plus simple sera de les téléverser dans un dossier images sur Hostinger et de remplacer les adresses src="...". Ça, faites-moi signe : c'est le genre d'étape où je peux vous aider efficacement.

6. Bon à savoir

Les modifications faites directement sur Hostinger ne sont pas recopiées automatiquement dans la sauvegarde GitHub. Ce n'est pas grave. De temps en temps (après plusieurs changements), vous pouvez me demander : « resynchronise la sauvegarde GitHub » — et je m'en occupe.
